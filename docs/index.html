Malnati/ops-eslint

Containerized ESLint with Flat Config for consistent linting across TypeScript, React, and NestJS projects.

Repository: github.com/Malnati/ops-eslint

How It Works

  1. Build or reuse a tooling image with ESLint + plugins.
  2. Mount target project path inside container.
  3. Run ESLint using central eslint.config.mjs.
  4. Write JSON report to .eslint/eslint-report.json.
  5. Expose outputs: status, error_count, warning_count, report_path.

Key Inputs

Input Default Description
path . Path to lint inside repository
fix false Enable ESLint auto-fix mode
fail_on_error true Fail action when lint returns non-zero
report_formatter json Formatter used to write lint report

Usage

- name: Run ops-eslint
  uses: Malnati/ops-eslint@v1
  with:
    path: src
    fail_on_error: true
